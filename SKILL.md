# Bookmark Organize B

## 作用

用于完整的书签整理流程（强调用户确认节点）：
- 输入：Edge/Chrome 导出的书签 HTML。
- 中间产物：带时间戳的 CSV（先给用户复核）。
- 最终产物：用户确认后，再输出可导入浏览器的书签 HTML。

## 不可跳过的规则

1. 不能跳过“分类体系确认”步骤。
2. 用户未确认 CSV 前，不能生成最终 HTML。
3. 不删除链接；仅对完全相同 URL 去重。
4. 输出文件必须带时间戳，避免覆盖。
5. 无法判别的链接必须访问页面再确认分类。

## 标准流程（严格顺序）

### 第 1 步：收集输入
- 获取源 HTML 文件路径。
- 确认输出目录（默认当前工作目录）。
- 完整读取源 HTML。

推荐话术：
- “我先读取你的书签 HTML，提取标题和 URL，先给你一版大小类草案确认，再逐条整理。”

### 第 2 步：解析与标准化
- 解析全部 `<A HREF=...>` 书签项。
- 提取：标题、URL、原目录路径（如有）、原描述（如有）。
- URL 原样保留，不做改写。
- 仅按“完全相同 URL”去重（默认保留首次出现）。

### 第 3 步：先给分类草案再确认
- 基于以下信息提出大类/小类草案：
  - 域名语义
  - 标题语义
  - 网站实际用途
- 若需要新增分类，明确说明新增原因。
- 等用户确认后，才进入逐条落位。

推荐话术：
- “我先给你分类体系草案（大类/小类）确认，确认后再逐条归类。”

### 第 4 步：逐条分类
- 按确认后的体系逐条归类。
- 同域名/同内容类型保持一致。
- 不确定时：
  1) 先看域名+标题+路径，
  2) 仍不明确则访问链接确认页面用途。

### 第 5 步：输出带时间戳 CSV
- 命名格式：`<原文件名>_<YYYYMMDD_HHMMSS>.csv`
- 示例：`favorites_2026_3_25_20260325_142233.csv`
- 列顺序固定：
  - `大类,小类,网站标题,备注说明,网址`
- `备注说明`：中文一句话，描述网站核心用途。

### 第 6 步：用户复核门禁（必须）
- 明确提示用户用 VSCode 或 Excel 打开 CSV 核对。
- 等用户明确回复“确认无误”后再继续。

推荐话术：
- “CSV 已输出，请先用 VSCode/Excel 检查分类；你确认后我再生成最终 HTML。”

### 第 7 步：生成最终书签 HTML
- 将确认后的 CSV 转回 Netscape Bookmark HTML 格式。
- 层级结构：`大类 -> 小类 -> 书签`。
- 命名格式：`<原文件名>_<YYYYMMDD_HHMMSS>.html`
- 如用户要求“收藏夹栏直入”，可额外生成 `PERSONAL_TOOLBAR_FOLDER="true"` 版本。

### 第 8 步：给出导入说明
- Edge/Chrome 导入步骤：
  1) 打开书签管理器
  2) 选择“从 HTML 导入书签”
  3) 选择生成的 HTML 文件
- 提醒：
  - 浏览器可能默认导入到“其他收藏夹”
  - 用户可手动拖到“收藏夹栏/书签栏”

## 分类原则

- 以“网站实际用途”优先，不依赖原目录名称。
- 重点区分：
  - AI：工具 / 教程 / 开发平台
  - 开发：官方文档 / 教程 / 开源仓库
  - 社区：技术博客 / 文章 / 论坛讨论
  - 学习 vs 娱乐（课程内容应归学习）
  - 设计相关站点应归设计大类
- `工具/通用资源` 仅作最后兜底。

## 建议默认大类

- `AI`、`开发`、`学习`、`设计`、`社区`、`工具`、`工作`、`娱乐`、`生活`

## 质量检查清单

CSV 输出前：
- 列顺序是否完全正确。
- 是否仅按“完全相同 URL”去重。
- 是否误删非重复链接。
- 备注是否非空且为一句话。
- 文件名是否带时间戳。

HTML 输出前：
- 用户是否已确认 CSV。
- HTML 是否为 Netscape 书签格式。
- 层级是否与确认分类一致。
- 是否已提供导入说明。

## 友好沟通节奏

建议阶段性更新：
- “正在读取与去重”
- “正在给分类草案”
- “正在逐条归类”
- “CSV 已输出，请先确认”
- “开始生成最终 HTML”

有不确定项时要说明：
- “有 N 条不确定，我会逐条访问页面确认后再归类。”
